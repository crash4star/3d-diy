# Telegram-бот «3D DIY Помощник ЖК»

MVP бота для локального сообщества по 3D-печати и DIY. Бот проводит пользователя
через форму заявки, показывает предпросмотр, публикует карточку в заданной теме
Telegram и сохраняет активные заявки в SQLite.

## Возможности MVP

- `/start`, `/rules`, `/order`, `/myorders`, `/myworks`, `/where`, `/cancel`;
- постоянное кнопочное меню для основных действий без ручного ввода команд;
- пошаговая форма заявки с текстом, документом или фотографией модели;
- переход из групповой команды `/order` в приватную форму (совместимо с Privacy Mode);
- приветствие новых участников с кнопкой подтверждения правил;
- сохранение подтверждения правил для повторного входа в группу;
- предпросмотр перед публикацией;
- публикация в заданную группу и тему;
- кнопки «Откликнуться» и «Закрыть заявку»;
- сохранение нескольких откликов без автоматической смены статуса;
- список кандидатов в разделе «Мои заявки» и выбор одного исполнителя автором;
- уведомление выбранного исполнителя и остальных кандидатов о решении;
- статусы выполнения «В работе» → «Готово» → «Завершено»;
- только выбранный исполнитель может отметить готовность, завершает заказ автор;
- личный раздел «Мои работы» со всеми откликами и доступными действиями;
- только автор может закрыть заявку;
- личный отклик исполнителя автору без публикации контактов в группе;
- восстановление активных заявок после перезапуска.

Незавершённая форма хранится в памяти и после перезапуска начинается заново. Сами
опубликованные заявки хранятся в SQLite.

## Подготовка Telegram

1. Создайте бота через [@BotFather](https://t.me/BotFather).
2. Добавьте его в группу и разрешите отправлять сообщения в тему заявок.
3. Узнайте ID группы и ID темы `Нужна 3D-печать` командой `/where`.
4. Каждый автор заявки должен один раз открыть личный чат с ботом и нажать
   **Start** — иначе Telegram не позволит боту прислать ему личный отклик.

Для MVP боту не нужны права на добавление администраторов или изменение данных
группы. Права удаления сообщений и ограничения участников понадобятся позднее,
при подключении модуля модерации.

### Мягкий онбординг участников

Telegram передаёт ботам служебные сообщения о вступлении даже при включённом
Privacy Mode. Когда в группу входит новый пользователь, бот:

1. публикует краткие правила;
2. показывает персональную кнопку **«Правила принимаю»**;
3. разрешает нажать её только вступившему участнику;
4. сохраняет подтверждение в SQLite;
5. при повторном входе приветствует пользователя без повторного подтверждения.

В текущем мягком режиме бот никого не ограничивает и не удаляет. Для приветствия
достаточно, чтобы бот состоял в группе и мог отправлять сообщения. Права удаления
сообщений и ограничения участников пока не требуются.

## Локальный запуск

Требуется Python 3.11 или новее.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'
cp .env.example .env
```

Для первого запуска укажите токен, а ID временно оставьте нулевыми:

```dotenv
BOT_TOKEN=токен-от-BotFather
TARGET_CHAT_ID=0
ORDERS_TOPIC_ID=0
```

Запустите бота, откройте тему `Нужна 3D-печать` и отправьте в ней `/where`.
Бот покажет готовые строки `TARGET_CHAT_ID` и `ORDERS_TOPIC_ID`. Остановите бота,
вставьте эти строки в `.env` и запустите снова.

Полный `.env` выглядит так:

```dotenv
BOT_TOKEN=токен-от-BotFather
TARGET_CHAT_ID=-1001234567890
ORDERS_TOPIC_ID=123
DATABASE_PATH=data/bot.db
ADMIN_IDS=123456789,987654321
LOG_LEVEL=INFO
BACKUP_DIR=data/backups
BACKUP_RETENTION_COUNT=14
```

Если группа работает без тем, задайте `ORDERS_TOPIC_ID=0`.

Запуск:

```bash
diy-bot
```

Проверки:

```bash
ruff check .
pytest
```

## Docker

```bash
cp .env.example .env
docker compose up --build -d
docker compose logs -f bot
```

База хранится в каталоге `data/`, подключённом к контейнеру как постоянный том.

## Резервное копирование

Команда `diy-bot-backup` создаёт согласованную копию работающей SQLite-базы через
Online Backup API, проверяет её командой `PRAGMA integrity_check`, выставляет права
`600` и оставляет только заданное количество последних файлов.

Ручной запуск в Docker:

```bash
docker compose run --rm --no-deps bot diy-bot-backup
```

Копии сохраняются в `data/backups/`. По умолчанию хранятся последние 14 файлов.
Настройки:

```dotenv
BACKUP_DIR=data/backups
BACKUP_RETENTION_COUNT=14
```

Для ежедневного запуска на VPS установите cron-файл из репозитория:

```bash
sudo cp deploy/3d-diy-backup.cron /etc/cron.d/3d-diy-backup
sudo chmod 644 /etc/cron.d/3d-diy-backup
```

Задание выполняется ежедневно в `00:15 UTC` (`03:15 Europe/Moscow`) и пишет журнал
в `/var/log/3d-diy-backup.log`.

Проверка последних копий:

```bash
ls -lh data/backups
tail -50 /var/log/3d-diy-backup.log
```

### Восстановление

Перед восстановлением остановите бота и сохраните повреждённую базу отдельно:

```bash
cd /opt/3d-diy
docker compose stop bot
mkdir -p data/restore-quarantine
mv data/bot.db* data/restore-quarantine/
cp data/backups/ИМЯ_КОПИИ.sqlite3 data/bot.db
chown 10001:10001 data/bot.db
chmod 600 data/bot.db
docker compose up -d bot
```

Локальные копии на том же VPS защищают от логической порчи и случайного удаления,
но не от потери всего сервера. Для полной схемы восстановления их следует позднее
дублировать во внешнее хранилище.

## Структура

```text
src/diy_bot/
├── backup.py        # проверяемые SQLite-бэкапы и ротация
├── config.py        # переменные окружения
├── handlers.py      # команды, FSM и кнопки Telegram
├── models.py        # модели заявки
├── presentation.py  # тексты карточек и клавиатуры
├── repository.py    # SQLite-хранилище
├── states.py        # состояния формы
└── main.py          # запуск polling
```

## Данные и безопасность

Бот хранит Telegram ID и отображаемое имя автора, поля заявки, статус, Telegram ID
опубликованного сообщения и техническое время создания. Для откликов сохраняются
Telegram ID, отображаемое имя, username и время выбора. Для подтверждения правил
также хранятся ID группы, Telegram ID, отображаемое имя и время принятия. Номер
квартиры и телефон не запрашиваются. Токен читается из `.env`, который исключён
из Git.

Резервные копии создаются штатной командой SQLite Online Backup, поэтому учитывают
WAL и не требуют остановки бота.
